It's pretty simple, dumbo. You POST a file name to http://what.seriesepisode.win and the service returns a json string with the parsed TV show name, series and episode (if it can do so). Just post the name using the name field name in the data. I even provided a CURL example at the top of the page so you wouldn't have to read below the fold. What kind of a programmer are you?

How?

It uses a pretty simple Python service and the Parse Torrent Name library to extract annoying file names into their constituent parts. It uses Twisted to wire up all the gubbins inside.

No, really, why?

My friends where all moaning about not being able to programmatically figure out what name, series and episode of a give TV show a file was. They wanted to be able to look up the shows using their own applications on APIs like TVMaze but no API they could find would let them grab the information without the filename being split into more useful parts. I thought I could do that, so I did.
